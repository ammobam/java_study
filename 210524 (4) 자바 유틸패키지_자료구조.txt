*선형구조

Dense List
	= 연접리스트(연속된 리스트)
	- array (배열)
	-- 크기가 고정됨. 메모리 낭비가 없음.
	-- 삽입/삭제시 복사작업 수행. 기존 데이터 삭제하고 교체.
	-- numpy. ndarray. 파이썬보다 연산속도가 빠름

Array List
	= 배열리스트 (배열이 아님!)
	-- vector 자료형
	-- 여분의 공간을 가진 배열. 메모리 낭비가 큼.
	-- 메모리 할당 방법? 여분의 공간이 maximum이 되도록.	(운영체제의 메모리 할당 방식)(워..트?)
	-- 삽입/삭제시 복사작업 수행. 밀어내기.	

Linked List
	- python의 list
	= 연결리스트
	- data와 link로 구성. link를 기억.
	-- link : 다음 데이터의 위치
	-- link(point)만 잇고 끊으면 돼서 삽입/삭제 편리함
	-- 메모리 낭비, 접근속도 느림
	-- 실제 data의 삭제가 아니기 때문에 복구가 가능함
		-- 복구 시간이 오래 걸리는 이유는 링크 연결 때문
	-- 예시: file의 저장 방식
		-- 1mb 파일 100개 삭제가 1gb 삭제보다 느림
		-- 링크 100개를 끊어내야 하기 때문

	- single linked list
		-- 접근 포인트가 1개

	- double linked list
		-- 파이썬의 list 자료구조
		-- 접근 포인트가 head, tail 2개

		- Tree(방향성), Graph(무방향)
		- Tree : 부모->자식 방향으로 연결. 가장 큰 것을 먼저 가리킴.
		-- 예: 대상 판단 시 이진분류의 정확도가 높음.

		- Graph : 시블링끼리도 연결 가능. 다시 돌아올 수 있음.
		-- 예: 바둑, 네비게이션. 최적경로 찾기. 임계경로.
Stack
	= LIFO (Last In First Out, 라이포)
	- 함수 부를 때 stack에 저장
	- 재귀함수가 느린 이유
	-- 몬테카를로 서치
	-- stack 안의 stack 안의 .... 느림
	-- 알바고는 하드웨어 성능으로 커버한 것
	- 이걸 개선. hadoop 관계형 데이터베이스보다 빠름.

Queue
	= FIFO (First In First Out)
	- 순차적으로 실행됨
	-- 컴퓨터 명령 실행구조
	-- 각 작업의 sink가 맞아야 효율적임.
	-- 예: 마스크 만드는 공정에서 필터부분 만드는 시간과 귀걸이 부분 만드는 시간이 같아야
	-- 딜레이 없이 다음 공정으로 넘어감.

Deque
	= 데크.
	-- 스크롤. 웹뷰. 버스 문 2개.

*Hashing
선형자료구조를 개선하기 위해 Hashing 이용한 자료구조 set, map.
- Hashing : 저장/검색하는 함수
-- 모든 데이터의 접근속도가 동일함
-- Hashing보다 빠른 방식은 현재 없음

Set	- python의 set(집합)
	-- 각 데이터에 %3해서 나머지 0,1,2에 따라 데이터 분류해서 저장함
	-- 나머지(0, 1, 2)로 데이터 부름
	-- *

	- Set은 데이터를 중복 없이 저장함
	- 데이터의 접근 속도가 빠름. 저장할 때 해싱을 이용하기 때문.

	- java는 HashSet, TreeSet, LinkedHashSet를 제공함
	-- HashSet: 일반적인 Set과 동일함. 저장순서 상관없이 저장.
	-- TreeSet : java기준대로 데이터 정렬함
	-- LinkedHashSet : 저장 순서대로 정렬함

Map	- python의 dict
	-- 나머지 0,1,2에 이름 붙이면 딕셔너리
	-- key, value (key는 문자열로)
	-- **

	- Map은 key-value를 쌍으로 저장함
	- key는 Set으로 구성되기 때문에 중복된 key를 이용해서 저장하면 데이터가 수정됨
	-- 이 방식으로 만들어진 데이터베이스가 NoSQL임 (KeyStore구조)
	-- HashMap, TreeMap, LinkedHashMap

	RandomTest.java 참고
	===============

	- 알 수 없는 숫자. 예측할 수 없는 데이터.
	- 프로그래밍 언어가 랜덤을 만드는 방법?
	- seed(난수표 번호)로 난수 모임을 생성하고 순서대로 숫자를 리턴하는 방식

	- seed를 랜덤하게 설정하면 정말 난수
	-- seed값을 랜덤하게 설정하는 가장 쉬운 방법은 현재 시간은 seed로 설정하는 것
	-- java나 python은 seed를 설정하지 않으면 랜덤한 seed 설정됨
	-- 파이썬에서 모델 만들 때 random_state라는 매개변수로 seed 설정하는 경우가 많음

	-- 게임이나 데이터분석 분야에서 특별한 경우가 아니면 seed는 고정시켜 놓고 작업함
	-- 예: 바다이야기 - 실제 확률과 체감 확률 / 고래가 나와야 배당률 104%인 게임. 리셋.

	- 과적합 모델을 방지하기 위해 Random 이용
	-- 과적합 : 훈련모델에 과적합하여 실제 데이터의 분석에 오차가 큰 경우
	-- 알고리즘 검증은 고정된 환경을 세팅해서 사용함
	-- 두 모델 비교할 때는 같은 환경 설정을 위해 같은 seed 이용함
