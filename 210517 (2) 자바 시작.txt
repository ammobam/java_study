**Data Type


1. Data 분류

1) 변경 가능 여부에 따른 분류
- mutable data : 변경 가능한 데이터, 변수
- immutable data : 변경 불가능한 데이터, 상수, 계산해야할 값
	- 데이터 분석시, 원본 데이터는 변경해선 안 됨
	-- 분석가는 데이터 copy해서 씀.
	-- copy 기능: R, Python의 Numpy/Pandas
	- 동시에 작업해야 할 데이터는 immutable로 줘야함 (은행, 게임)

2) 데이터 개수에 따른 분류
데이터 1개를 저장하는 것
	- Scala Data = Value Type
	- 데이터를 가리키는 이름이 데이터 자체
0개 이상의 데이터를 저장하는 것
	- Vector Data = Collection = Refernce Type
	- 데이터가 여러개이기 때문에 이름은 데이터에 접근하기 위한 참조가 됨
	- . 또는 [인덱스]를 이용해서 세부 데이터 사용

3) 데이터 모양에 따른 분류
- 정형 데이터 : 모양이 일정, 테이블 구조
- 비정형 데이터 : 모양이 일정하지 않아서 테이블 구조로 만들 수 없는 데이터. (예: 음성, 일반 텍스트 등)
- 반정형 데이터 : 비정형처럼 보이지만 데이터를 해석할 수 있는 메타 데이터가 포함된 데이터 (예: JSON, XML 등)

4. 컴퓨터가 데이터를 저장하는 방법

- 컴퓨터는 데이터를 2진수로 저장함

1) 양의 정수를 저장하는 방법
- 10진수를 2진수로 변경해서 저장
	입력 	int a = 17;
	공간할당	a => 00000000 00000000 00000000 00000000 (int는 5byte)
	연산	17/2 : 	8, 1		몫, 나머지
		8/2 : 	4, 0
		4/2 : 	2, 0		몫이 0이 될 때까지 반복
		2/2 :	0, 0
	값 저장 	a => 00000000 00000000 00000000 00001000
					나머지를 역순으로 기재

2) 음의 정수를 저장하는 방법
- 양수의 2의 보수를 구해서 저장
1의 보수(0->1, 1->0)
2의 보수 = 1의 보수 + 1

	입력 	int a = -17;
	양수계산 	17 => 	00000000 00000000 00000000 00001000
	1의 보수		11111111 11111111 11111111 11110111
	2의 보수		11111111 11111111 11111111 11111000
					2는 자릿수 올라감
					이게 -17이 저장된 방식


- 2진수를 10진수로 읽기
-- 양수
-- 음수인 경우
	- 맨 앞자리 값이 1이면 음수로 판단
	- 1의 보수를 변경(1->0, 0->1)
	- 1의 보수에 1 더해서 2의 보수 만들어주고
	- 이진법 양수 읽을 때처럼 읽음
	- 2의 보수에 음수 취해주기

3) -1 또는 음수
'-1'은 음수 1일 때도 있지만 다른 의미로 사용되는 경우가 있음
	1을 2진수로
	00000000 00000000 00000000 00000001
	-1을 2진수로
	11111111 11111111 11111111 11111111
	-- 음수가 필요 없는 경우 -1은 가장 큰 숫자임
- find(), indexOf() 등과 같은 함수는 문자열 또는 배열에서 데이터의 위치를 찾아줌
- 이 함수들은 데이터를 찾지 못한 경우 -1 또는 음수를 return함
- 파이썬의 경우 index[-1]하면 가장 "마지막" 수를 찾아줌


4) 실수 저장

5) 문자 저장

1) Overflow
- 자신의 범위를 넘어서는 데이터를 강제로 저장해서 가장 작은 숫자부터 다시 시작하도록 하는 것

		short s = (short)32768;
		System.out.println(s); 	// -32768

2) Underflow
- 자신의 범위를 아래 방향으로 넘어서는 데이터를 강제로 저장해서 가장 큰 숫자부터 다시 시작하도록 하는 것

		s = (short)-32769;
		System.out.println(s);	// 32767

**Operator
1.용어
- Operator : 연산을 수행해주는 부호나 명령어
- Operand : 연산에 사용되는 데이터
- Expression : 연산자와 피연산자를 이용하여 연산 과정을 기술한 것

2. 연산의 분류
1) 연산에 사용하는 데이터나 결과에 따른 분류
- 산술 연산: 숫자데이터를 연산하여 숫자데이터가 return되는 연산
- 논리 연산: 숫자데이터나 논리데이터(boolean)를 연산하여 논리데이터가 return되는 연산

2) 피연산자의 개수에 따른 분류
- 단항 연산 (Unary) : 데이터가 1개만 있으면 수행되는 연산
- 이항 연산 (Binary) : 데이터가 2개 있어야 수행되는 연산
- 삼항 연산 (Ternary) : 데이터가 3개 있어야 수행되는 연산

3. 최우선 연산자
1) ( ) : 우선순위 부여
2) [ ] : 배열의 크기와 인덱스(첨자)를 나타낼 때 사용하는 연산자


4. 단항 연산자
1) '~' 연산자 : 1의보수(0->1, 1->)를 구하는 연산자
- 양수의 1의 보수는 부호는 음수가 되고 절대값은 1증가
- 음수의 1의 보수는 부호는 양수가 되고 절대값은 1감소
- 숫자 자체 의미는 없고, 색상 반전에 이용
-- 카메라 필름 -> 디지털사진 변환 등

2) '!' : boolean 데이터에만 사용할 수 있는 연산자로, true<->false
	if(flag == false)
	if(!flag == true) 	// 같은 코드
	if(!flag)	// == true는 생략 가능

3) +, -
'+': 양수
'-': 음수. 정확히는 2의보수.

4) ++, --
- 정수 변수에만 사용 가능함
- '++' 정수 변수의 값을 1 증가
- '--' 정수 변수의 값을 1 감소
- 데이터 앞/뒤에 위치할 수 있음
- 명령문과 함께 사용하면 앞에 있을 때는 값 증감을 한 다음 명령문에 사용함
- 명령문과 함께 사용하면 뒤에 있을 때는 저장된 데이터 값으로 명령문 실행한 다음 증감

		int x = 0;
		int result = ++x + ++x;
		System.out.println(x);
	# Java는 3
	# C언어는 4 // 증감 반영 후 ++ 뗀 n을 넣고 다시 연산
	- Java와 C언어의 비교
	-- Java는 메모리는 적게 쓰고 연산은 느림
	-- C언어는 메모리를 많이 쓰지만 연산이 빠름

6) shift

- 정수 데이터에서만 사용함
데이터 << 비트수 : 데이터를 왼쪽으로 비트수만큼 shift
	-- 왼쪽으로 n비트 이동하면 값이 2^n배 됨
	-- 부호(첫번째 비트에는 적용하지 않음)는 변경하지 않음
	10<<2 	// 10을 32비트 분해해서 왼쪽으로 2비트 이동함
		//40

데이터 >> 비트수 : 데이터를 오른쪽으로 비트수만큼 shift
	-- 오른쪽으로 n비트 이동하면 값이 2^(-n)배 됨
	-- 부호(첫번째 비트에는 적용하지 않음)는 변경하지 않음
	10>>2	// 10을 32비트 분해해서 오른쪽으로 2비트 이동함
		// 2	// int라서 소수점 아래 버림
데이터 >>> 비트수 : 데이터를 첫번째 비트부터 오른쪽으로 비트수만큼 shift
	-- 부호변경 가능, 부호를 결정하는 첫번째 칸이 무조건 밀림

- 정수는 32비트로 만들어지기 때문에 32번 이상 shift할 수 없음
-- 32번 이상 shift하면 32로 나눈 나머지만큼만 shift 실행됨


- 윈도우 프로그래밍 시 '단축키' 만들 때 사용함

	int k = 10;
	System.out.println("k<<2 : " + (k<<2));
		// 연산되는 부분 k<<2를 ( )로 묶지 않으면 에러 남

-- 선택 옵션을 만들 때 문자열을 입력하는 방식이 있고, 정수 중에 하나를 선택하는 방식이 있음
