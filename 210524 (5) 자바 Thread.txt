- 스레드는 작업 도중에라도 자신의 작업을 중지하고, 다른 작업을 수행할 수 있게 함

- 멀티 스레드 이용시 발생할 수 있는 문제점

1. 상호배제 (Mutual Exclusion)
	-- 하나의 스레드가 사용 중인 자원을 다른 스레드가 수정하지 못하도록 해야 함
	-- lock으로 해결
	-- 과거에는 lock을 synchronized(동기화) 라고 부름
	- java에서는 synchronized 메소드나 블럭을 이용하여 특정 영역에서 코드를 동시에 수행하지 않도록 함
	-- 해결1. synchronized 메소드
	-- 해결2. synchronized block (권장함!)
		-- 이 영역 내에서는 this를 같이 사용할 수 없음

2. 생산자와 소비자 문제
	- 생산자와 소비자 스레드가 동시에 수행되는 경우
	- 생산자 스레드가 생산을 한 다음 소비자 스레드가 동작하도록 하는 문제

	- notify로 해결
	-- 생산자스레드는 생산만, 소비자스레드는 소비만 할 때 이 스레드들은 동시 수행돼도 됨
	-- 하지만 생산자가 자원을 생산하지 않은 상태에서 소비자스레드가 자원을 사용할 때 NullPointerException이 발생함

	- 해결방법
	--1) 소비자스레드는 자원을 사용하기 전에 자원의 존재여부를 확인
	--2) 자원이 없는 경우 wait을 이용해서 대기함
	--3) 생산자 스레드는 자원을 생성하면 notify를 통해 wait상태인 스레드를 깨워서 동작함

3. Dead Lock 문제
	- 결코 발생할 수 없는 사건을 '무한정 대기'하는 것
	-- 2개의 스레드가 2개 이상의 자원을 사용해서 작업을 처리해야 하는 경우
	-- 하나의 스레드가 하나의 자원을 갖고 있고 다른 스레드가 다른 자원을 가지고 있어서
	-- 자원을 양보하지 않으면 스레드가 종료되지 못하는 무한 대기 상태
	- 동기화 되어 있는 블럭 내에서 다른 동기화를 중복해서 발생하는 문제임

4. 무한 루프 문제
	- Map의 여러 개의 스레드에 데이터를 동시에 put하면 무한 루프가 발생함
	-- 하나의 데이터베이스에 2개 접속하여
	-- 데이터 삽입/수정/삭제한 상태에서 commit하지 않은 경우
	-- 다른 곳에서 데이터 삽입/수정/삭제 요청하면 무한 루프에 빠짐
		-- 읽기 Shared Lock 문제
		-- 쓰기 Exclusive Lock 문제
